# PM Interview 1h Rescue Output Template

Use this template to generate the final answer. Adapt section length to time left and input richness.

## 0. One-Page Emergency Summary

Place this at the beginning when the output is long or the interview is within 30 minutes.

Include:

- 面试定位: “这场你要把自己打成 {X型产品候选人}。”
- 三个卖点: one sentence each.
- 三张王牌经历: name + one-line use case.
- 必背三问: self-intro, why this role, strongest project.
- 风险防守: top 2 risks + one-line defense.
- 反问: top 2 interviewer questions.
- Last instruction: “最后10分钟不要再看长文，只看本页。”

## 1. 面试胜率判断

Give a direct but constructive diagnosis:

- Current fit: 高/中高/中/偏低.
- Best leverage: the single improvement most likely to raise pass probability.
- Biggest risk: the issue most likely to cause rejection.

Example:

> 这场面试你的基础匹配度是中高。JD最看重增长策略和数据拆解，而你的简历里有相关项目，但现在的问题是表达像“做活动”，不像“做产品增长”。这1小时最该做的不是补知识，而是把项目讲成“发现问题—拆指标—设计方案—验证结果”的产品闭环。

## 2. 本岗位真实招聘意图

Structure:

1. 岗位类型判断.
2. 团队可能要解决的问题.
3. 面试官最可能考察的能力.
4. 面试官最担心的候选人缺陷.
5. JD背后的隐藏问题.

Useful sentence patterns:

- “这个岗位表面写的是 {JD wording}，但真实考察更像是 {ability}。”
- “这不是一个泛产品岗，更偏 {role archetype}。”
- “面试官会怕候选人只会讲想法，不能证明 {metric/landing/collaboration}。”

Create:

- One-line positioning.
- Three narrative pillars.
- One “do not position as” warning.

Format:

| 项目 | 内容 |
|---|---|
| 推荐定位 | {X型产品候选人} |
| 主线1 | {能力 + 证据} |
| 主线2 | {能力 + 证据} |
| 主线3 | {能力 + 证据} |
| 不要打成 | {weak or wrong positioning} |

## 4. JD-简历匹配矩阵

Required columns:

| JD要求 | 简历证据 | 匹配度 | 面试表达建议 | 风险/补救 |
|---|---|---|---|---|

Rules:

- Use 高/中/低 fit labels.
- Mark “证据不足” directly when needed.
- Convert resume bullets into interview claims.
- Provide compensation strategy for weak areas.

Example expression:

> 不要说“我负责了用户增长活动”，要说“我先看新用户从注册到首个关键行为的漏斗，发现主要流失发生在价值感知阶段，所以设计了低成本引导和激励机制。”

## 5. 三张王牌经历

For each experience:

### 王牌经历 {n}: {project name}

- **适合回答**: list 3-6 questions.
- **对应JD能力**: list mapped capabilities.
- **一句话版本**: one concise answer.
- **STAR+R结构**:
  - S: situation/background.
  - T: task/objective.
  - A: actions with PM thinking.
  - R: result, metrics, learning.
  - +R: reflection, limitation, next iteration.
- **必须强调**: details that prove PM capability.
- **不要主动说**: weak details, irrelevant execution, unverified claims.
- **可能追问**: 3-5 follow-ups.
- **追问防守**: concise response strategy.

PM-oriented STAR+R must emphasize:

- Problem definition.
- User/business insight.
- Prioritization logic.
- Collaboration and tradeoffs.
- Measurement and iteration.

## 6. 高频问题预测与答案

Group questions:

### 必考题

Usually include:

- 请做一个自我介绍.
- 介绍一个你最有代表性的产品/项目经历.
- 为什么投这个岗位/公司?
- 你为什么适合这个岗位?

For each:

- 面试官真正想听什么.
- 回答结构.
- 推荐话术.
- 避坑提醒.

### 高概率题

Choose based on JD:

- 需求优先级怎么判断?
- 指标下降怎么分析?
- 如何理解目标用户?
- 如何推动研发/运营/设计协作?
- 你做过最困难的项目是什么?
- 你怎么做竞品分析?
- 如果让你优化某指标，你怎么做?

### 风险题

Choose based on resume gaps:

- 你这个项目是不是更偏运营?
- 你没有完整产品实习，为什么能做PM?
- 这个数据怎么归因?
- 你在项目里到底负责什么?
- 为什么转产品?
- 为什么离开上一段经历?
- 你的缺点是什么?

### Case题

Provide a role-specific case framework, not a full generic textbook.

## 7. 自我介绍与动机话术

Generate:

### 30秒自我介绍

Structure:

> 我是谁 + 最相关经历 + 核心能力标签 + 和岗位匹配点

### 90秒自我介绍

Structure:

> 背景 + 2-3段代表经历 + 能力沉淀 + 为什么匹配该岗位

### 为什么投这个岗位

Use four layers:

1. Company/business interest.
2. Role responsibility match.
3. Personal experience fit.
4. Growth motivation.

Avoid vague praise like “平台大、发展好、我很喜欢产品”.

## 8. PM专属临场框架

Provide only the most relevant framework for the JD.

Format:

- Applicable scenario.
- 5-7 step answer structure.
- One sample opening sentence.
- One metric checklist.
- One common pitfall.

## 9. 反问面试官问题

Generate 3-5 questions. Tie them to the role.

High-signal examples:

- 这个岗位入职后最核心的业务目标是什么?
- 团队目前最希望这个角色解决的问题是什么?
- 如果我加入，前三个月您最期待我在哪些方面产生价值?
- 这个岗位更偏策略探索，还是更偏需求落地和项目推进?
- 团队判断一个PM表现好的核心标准是什么?

Avoid:

- 薪资福利.
- 可以学到什么.
- 公司发展怎么样.
- Vague culture questions.

## 10. 最后10分钟速记页

Make this short enough to screenshot.

Required:

1. 30秒自我介绍.
2. 三个卖点.
3. 三张王牌经历.
4. 两个风险防守.
5. 三个反问问题.
6. One calming instruction.

Example calming instruction:

> 能过简历筛选，说明你已经有基本胜任信号。现在不是证明自己完美，而是把最匹配的证据讲清楚。
