# Time Modes for PM Interview Rescue

Choose the mode based on time left. If user does not specify time, default to 60-minute mode but include a final 10-minute page.

## 60-Minute Mode

Goal:

- Build the full battle plan.

Recommended allocation:

1. 10min: Understand JD and company/business.
2. 15min: Match resume to JD and select top 3 experiences.
3. 15min: Prepare must-ask answers and risk defenses.
4. 10min: Prepare role-specific PM case framework.
5. 5min: Prepare interviewer questions.
6. 5min: Calm down and read one-page notes.

Output:

- Complete rescue pack.
- Detailed but still skimmable.
- Include final one-page cram sheet.

## 30-Minute Mode

Goal:

- Prevent catastrophic underperformance and make the candidate sound matched.

Only output:

1. Interview positioning.
2. Top 3 selling points.
3. Top 3 experiences and which questions they answer.
4. Top 5 likely questions with concise answer scripts.
5. Top 3 risk defenses.
6. 30-second self-introduction.
7. Three interviewer questions.

Do not output:

- Long company research.
- Full PM framework explanations.
- More than 8 questions.

## 15-Minute Mode

Goal:

- Give the candidate a screenshot-ready page.

Only output:

- 30-second self-intro.
- One-line positioning.
- Three selling points.
- Three project keywords with one-line proof.
- Three must-answer question scripts.
- Two risk defenses.
- Two interviewer questions.
- One calming instruction.

Tone:

- Direct.
- Short.
- No theory.

## Less Than 10 Minutes

Goal:

- Reduce panic and align the first answer.

Only output:

1. 30-second self-intro.
2. Strongest project one-liner.
3. “Why this role” one-liner.
4. One risk defense.
5. One question to ask interviewer.

End with:

> 不要再补新知识。现在只要把“我是谁、我为什么匹配、我做过什么证据”讲清楚。

## Compression Rules

When shortening:

1. Keep positioning before details.
2. Keep scripts before analysis.
3. Keep risk defenses before optional questions.
4. Keep top 3 experiences before broad question lists.
5. Keep one-page notes at all costs.

## Escalation Rules

If the candidate’s resume-JD fit is weak:

- Spend more output on compensation strategy and honest framing.
- Avoid pretending the fit is strong.
- Emphasize transferable PM capabilities.

If the candidate has strong experience but poor expression:

- Spend more output on answer scripts and STAR+R.
- Reduce general PM knowledge.

If the JD is vague:

- Infer role type from available signals.
- State assumptions.
- Prepare universal PM questions plus role-adjacent case framework.
